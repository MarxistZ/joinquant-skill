# 14 — 策略引擎机制

> 聚宽回测/模拟引擎的核心机制：撮合、复权、滑点、税费、风险指标、运行频率。

## 撮合机制 ★

### 订单处理流程

```
订单创建 → 订单检查 → 报单 → 确认委托 → 撮合
                ↓（未通过）
              订单取消
```

- **回测**：全流程同步，市价单立即撮合
- **模拟交易**：同回测
- 所有未完成订单在当日结束后撤销

### 下单时的撮合（未启用盘口撮合）

| 条件 | 处理 |
|---|---|
| 最新价+滑点在涨跌停范围内 | 成交，成交价=最新价+滑点 |
| 买入 | 委托价 = 最新价 + 滑点 |
| 卖出 | 委托价 = 最新价 - 滑点 |
| 跌停时市价卖单 | 撤销 |
| 涨停时市价买单 | 撤销 |
| 限价单不满足条件 | 挂单等待 |

**成交量限制**（回测）：不超过当日总成交量 × `order_volume_ratio`

**最新价定义**：
- 开盘时刻 → 开盘价
- 其他时刻 → 上一分钟最后价格 / 上一 Tick 最新价

### 下单时的撮合（启用盘口撮合，仅模拟交易）

- 按对手盘逐档撮合，从买一/卖一开始
- 加权均价成交
- 不限制成交量
- 无盘口数据时回退到普通撮合

### 挂单撮合

**按分钟 Bar**（天/分钟频率策略）：
- 买入：委托价 ≥ Bar 最低价 → 成交
- 卖出：委托价 ≤ Bar 最高价 → 成交
- 成交价 = 委托价
- 成交量限制：不超过 Bar 成交量 × `order_volume_ratio`

**按 Tick**：
- 买入：委托价 ≥ Tick 最新价 → 成交
- 卖出：委托价 ≤ Tick 最新价 → 成交
- **不检查成交量**

### 非交易时段下单

- 市价单和限价单都会挂单
- 日频：开盘时撮合
- 分钟/Tick：下一个 Bar/Tick 完成时撮合

## 复权机制 ★

### 传统前复权模式（不推荐）

所有价格基于回测当天的复权因子计算。**存在未来函数偏差**。

### 真实价格（动态复权）模式 ★★★

```python
set_option('use_real_price', True)
```

回测到某天时，使用该天的真实价格和复权因子：
- 每到新的一天，发生拆合/分红时自动调整持股数量
- 调整后数量为小数时向下取整，`available_cash` 相应调整
- **不要跨日缓存** history/get_price 等 API 返回的价格

### 拆分合并与分红

| 模式 | 处理方式 |
|---|---|
| 传统前复权 | 历史价格全部前复权，不调整持仓 |
| 真实价格 | 按历史情况调整账户（现金/持股数量变化），有日志提示 |

### 股息红利税

| 持有时间 | 税率 |
|---|---|
| ≤ 1 个月 | 20% |
| 1 个月 ~ 1 年（含） | 10% |
| > 1 年 | 免税 |

回测/模拟中统一按 **20%** 税率计算（无法准确知道持仓时间）。

## 交易税费

包含券商手续费和印花税：
- **券商手续费**：双边收费，默认万三，最低 5 元
- **印花税**：卖方单边征收，默认千分之一

通过 `set_order_cost` 设置。详见 `references/01-strategy-setup.md`。

## 风险指标

### 更新时机

- 每天 17:00 更新一次
- 仅基于每日收盘收益计算，不考虑盘中
- 例外：分钟/Tick 模拟盘每分钟更新策略收益和基准收益

### 主要指标

| 指标 | 说明 |
|---|---|
| 年化收益率 | 年化后的累计收益 |
| Alpha | 策略超额收益（CAPM 模型） |
| Beta | 策略相对市场的敏感度 |
| Sharpe | 夏普比率，风险调整后收益 |
| Max Drawdown | 最大回撤 |
| 信息比率 | 主动收益/主动风险 |
| 波动率 | 年化收益标准差 |

## 运行频率

| 频率 | 执行次数 | 说明 |
|---|---|---|
| 天 | 每日 1 次 | 可获取任何粒度数据 |
| 分钟 | 每分钟 1 次 | 每天 240 次 |
| Tick | 每个 Tick 1 次 | 需要权限 |

### 运行时间顺序

| 时段 | 执行内容 |
|---|---|
| 09:00 | `run_xxx` 指定 `time='09:00'` 的函数 + `before_trading_start` |
| 09:30~15:00 | `run_xxx` 指定时间的函数 + `handle_data` |
| 15:30 | `run_xxx` 指定 `time='15:30'` 的函数 + `after_trading_end` |

**同一时间点**：先执行 `run_xxx` 指定的函数，然后是 `before_trading_start` / `handle_data` / `after_trading_end`。

### 重要注意

- `run_xxx` 和 `handle_data` 不要在同一策略中混用，建议统一用 `run_xxx`
- 空函数会降低运行速度，不需要的回调不要实现
- 日回测中 `handle_data` 在 09:30:00 运行，`data` 为昨天的天数据
- 分钟回测每天执行 240 次，不包括 11:30 和 15:00 这两分钟

## 安全限制

| 限制 | 说明 |
|---|---|
| 文件 | 只读指定 Python 库，不能写文件（除 write_file/read_file） |
| CPU/内存 | 有限制 |
| 网络 | 可访问，下载 500Kbps，上传 10Kbps |
| 超时 | `handle_data` 超过 30 分钟则停止 |
| 进程隔离 | 每个用户代码在独立进程中运行 |

---

## K 线数据说明

- 所有行情 K 线为**后对齐**（时间标记为数据结束时间）
- 分钟 K 线：从 09:31 开始，到 15:00 结束，共 240 根
- 09:31 的一分钟 K 线：数据时间为 09:25:00~09:30:59，开盘价是集合竞价价格

## 常见失败模式

❌ 不设 `use_real_price=True` → 回测结果不可靠
❌ 跨日缓存 `get_price` 返回的价格 → 不同日期前复权价格不同
❌ 以为风险指标实时更新 → 只有 17:00 更新
❌ `order_volume_ratio` 绕过成交量限制（多次下单） → 回测不真实
❌ 混用 `handle_data` 和 `run_daily` → 建议统一用 `run_daily`
