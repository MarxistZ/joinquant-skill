# 14 — 策略引擎机制

> 聚宽回测/模拟引擎的核心机制：生命周期函数、撮合、复权、滑点、税费、风险指标、运行频率、数据覆盖、模拟盘行为。

---

## 策略生命周期函数

| 函数 | 触发时机 | 说明 |
|---|---|---|
| `initialize(context)` | 开始回测/模拟时 | 整个生命周期执行一次 |
| `handle_data(context, data)` | 每个 Bar | 天频每天一次，分钟频每分钟一次；**Tick 频率不支持** |
| `handle_tick(context, tick)` | 每个 Tick | Tick 频率策略每个 Tick 调用一次；需先订阅标的 |
| `on_event(context, event)` | 持仓标的发生事件时 | 处理分红送股 (`DividendsEvent`) 和强行平仓 (`ForcedLiquidationEvent`) |
| `before_trading_start(context)` | 每日 09:00 | 开盘前执行 |
| `after_trading_end(context)` | 每日 15:30 | 收盘后执行 |
| `on_strategy_end(context)` | 回测/模拟正常结束时 | 清理操作 |
| `process_initialize(context)` | 每次进程重启时 | **模拟盘每天重启时执行** |
| `after_code_changed(context)` | 模拟盘代码修改保存后 | **修改全局变量在此处理** |

- 所有回调都是**可选**的。不需要就省略，写空函数会降低运行速度。
- `process_initialize` 和 `after_code_changed` 是模拟盘专用。
- `handle_tick` 是 Tick 频率策略专用；`on_event` 需持仓标的发生对应事件时才被调用。

---

## 运行频率

| 频率 | 执行次数 | 说明 |
|---|---|---|
| 天 | 每日 1 次 | 可获取任何粒度数据 |
| 分钟 | 每分钟 1 次 | 每天 240 次（不含 11:30 和 15:00 这两分钟） |
| Tick | 每个 Tick 1 次 | 需要向平台申请权限 |

### 运行时间点

| 时段 | 执行内容 |
|---|---|
| **09:00** | `run_xxx(time='09:00')` + `before_trading_start` |
| **盘中** | `run_xxx` 指定时间的函数 + `handle_data` |
| **15:30** | `run_xxx(time='15:30')` + `after_trading_end` |

**同一时间点执行顺序**：先 `run_xxx` 指定函数，然后 `before_trading_start` / `handle_data` / `after_trading_end`。

**日频 handle_data 特别说明**：在 09:30:00 执行，data 为**昨天**的天数据。

### 关于 run_xxx 与 handle_data

- 建议用 `run_daily`/`run_weekly`/`run_monthly` 替代 `handle_data`，可精确指定执行时间
- **不要混用** `run_xxx` 与 `handle_data`
- 可写多个 `run_xxx` 实现不同时间点的执行逻辑

## Bar 与数据说明

**Bar** 是一定时间段的 K 线数据单元：
- **日线 Bar**：一天内的分钟序列构成
- **分钟 Bar**：一分钟内的 Tick 序列构成

**K 线后对齐**：标识时间 = 数据结束时间。一分钟 K 线从 09:31 开始到 15:00 结束，共 240 根。09:31 的 K 线数据时间为 09:25~09:30，开盘价是集合竞价结果。

---

## 数据覆盖范围

- **股票**：2005 年以来所有 A 股行情、财务、融资融券数据（含已退市股票）
- **基金**：600+ 交易所上市基金（ETF、LOF、分级 A/B、货币基金）
- **金融期货**：中金所所有金融期货品种
- **指数**：近 600 种指数的行情及成分股数据
- **行业/概念板块**：支持按行业、板块、概念选股
- 行情数据已处理前复权信息
- 当日回测数据在 **T+1 00:01** 更新

## 订单撮合流程 ★

### 订单处理流程

```
订单创建 → 订单检查 → 报单 → 确认委托 → 撮合
                ↓（未通过）
              订单取消
```

- **回测**：全流程同步，市价单立即撮合
- **模拟交易**：同回测
- 所有未完成订单在交易日结束后（约 17:00）撤销
- 每日下单最大 **10000 笔**

### 下单时的撮合（未启用盘口撮合）

满足以下条件以「最新价 + 滑点」成交：
- **买入/开多/平空**：委托价 >= 最新价 + 滑点
- **卖出/开空/平多**：委托价 <= 最新价 - 滑点

**成交量限制**：
- 回测：不超过当日总成交量 × `order_volume_ratio`
- 模拟盘：全部成交

**超出部分**：市价单撤单，限价单挂单。

**涨跌停特殊处理**：跌停时市价卖单撤销；涨停时市价买单撤销。

**最新价定义**：
- 开盘时刻 → 开盘价
- 其他时刻 → 上一分钟最后价格 / 上一 Tick 最新价

### 下单时的撮合（启用盘口撮合，仅模拟交易）

需 `set_option('match_with_order_book', True)` 开启：
- 从买一/卖一开始逐档撮合
- 按成交量加权均价成交
- 不限制成交量
- 无盘口数据时回退普通撮合

### 挂单撮合（限价单未立即成交时）

**按分钟 Bar**（天/分钟频率策略）：
- 买入/开多/平空：委托价 **>** Bar 最低价则成交
- 卖出/开空/平多：委托价 **<** Bar 最高价则成交
- 成交价 = 委托价
- 成交量限制：
  - 回测：不超过本分钟 Bar 成交量 × `order_volume_ratio`
  - 模拟交易：不超过本分钟 Bar 成交量

**按 Tick**（Tick 频率策略）：
- 买入：委托价 >= Tick 最新价则成交
- 卖出：委托价 <= Tick 最新价则成交
- **不检查成交量**

### 非交易时段下单

市价单和限价单都会挂单。日频策略开盘时撮合，分钟/Tick 策略在下一个 Bar/Tick 完成时撮合。

## 动态复权 ★★★

传统前复权存在**未来函数偏差**（回测中各历史时刻的价格基于同一未来日期的复权因子）。**强烈建议开启**真实价格模式：

```python
set_option('use_real_price', True)
```

**开启效果**：
- 每天看到的价格是当天真实价格
- 下单、交易详情、持仓详情均为真实价格
- 数据 API（`get_price`/`history`/`attribute_history` 等）返回基于当天日期的前复权价格
- **不要跨日缓存**这些 API 返回的价格——不同日期前复权价格不同
- 持仓股票发生送股/分红时，股数/现金自动调整（调整后股数向下取整）

**限制**：
- 对**期货不生效**
- 对场内基金生效，但**不建议**含场内基金的策略开启（除权日披露不标准）

## 股息红利税

| 实际持有时间 | 税率 |
|---|---|
| <= 1 个月 | 20% |
| 1 个月 ~ 1 年（含） | 10% |
| > 1 年 | 免税 |

分次买入按**先进先出**原则计算持股时间。但**回测及模拟中统一按 20% 预扣**（无法预知卖出时间）。

---

## 滑点

模拟真实市场预期价与成交价的偏差，用 `set_slippage` 设置：

| 模型 | 说明 | 适用范围 |
|---|---|---|
| `FixedSlippage(0.02)` | 固定价差 | 通用 |
| `PriceRelatedSlippage(0.002)` | 百分比价差 | 默认(0.00246) |
| `StepRelatedSlippage(2)` | 最小变动价位跳数 | 期货专用 |

货币基金/场内 MMF 滑点恒为 0，不可修改。

A 股默认值（可通过 `set_order_cost` 修改）：

| 费用 | 比例 | 方向 |
|---|---|---|
| 券商佣金 | 万三（0.03%），最低 5 元 | 双边 |
| 印花税 | 千分之一（0.1%） | 卖方单边 |

## 风险指标

每天 **17:00 左右**更新一次，基于每日收盘收益计算。分钟/Tick 模拟盘每分钟更新策略收益和基准收益。

| 指标 | 说明 |
|---|---|
| 策略收益 | (期末净值 - 期初净值) / 期初净值 |
| 年化收益率 | 按 250 个交易日年化 |
| Alpha | 策略相对基准的超额收益（CAPM 模型） |
| Beta | 策略对基准的敏感度 |
| Sharpe | 夏普比率，单位总风险带来的超额报酬 |
| 最大回撤 | 策略可能的最糟糕亏损 |
| 胜率 | 盈利交易次数 / 总交易次数 |
| 盈亏比 | 总盈利额 / 总亏损额 |

更多指标（索提诺比率、信息比率、下行波动率等）详见平台回测结果页面。

## 模拟盘注意事项

| 特性 | 说明 |
|---|---|
| 系统延迟 | 模拟盘有约 10s 系统延迟，日志时间为逻辑时间（非实际时间） |
| 进程重启 | **每天结束后保存状态、结束进程**，第二天重启 |
| 状态序列化 | 用 pickle 序列化 `g` 和 `context`，**上限 30MB** |
| 不可序列化对象（数据库连接/文件句柄/网络连接等） | 需以 `__` 开头命名，并在 `process_initialize` 中初始化 |
| 单函数超时 | 每个函数运行时间不超过 **1800s（30 分钟）** |
| 进程内存 | 不超过 **3GB** |
| 代码修改 | 修改后执行 `after_code_changed`（不触发 initialize） |
| 动态复权 | **强烈建议模拟盘开启** |

### 模拟 vs 回测的微小差异

即使参数一致，以下原因仍会导致差异：
- Python 版本差异
- 策略中使用随机数、不稳定排序或 dict 遍历
- 回测中使用了未来数据
- 模拟盘每日重启涉及全局变量持久化（回测无此问题）
- 模拟盘可暂停，回测不支持

## 安全沙箱限制

| 限制 | 说明 |
|---|---|
| 文件 | 只读指定 Python 库，不能写文件（除 read_file/write_file） |
| 网络 | 可访问，下载 500Kbps，上传 10Kbps |
| CPU/内存 | 有限制（模拟盘 3G 内存上限） |
| 超时 | `handle_data` 超过 30 分钟则停止 |
| 进程隔离 | 每个用户代码在独立进程中运行 |
| 加密 | 策略代码加密存储在数据库，全程 HTTPS |

## 常见失败模式

- 不设 `use_real_price=True` — 回测结果不可靠
- 跨日缓存 `get_price` 等结果 — 不同日期前复权价格不同
- 以为风险指标实时更新 — 只有 17:00 更新
- 多次下单绕过 `order_volume_ratio` 限制 — 回测不真实
- 混用 `handle_data` 和 `run_daily` — 建议统一用 `run_xxx`
- 模拟盘使用不可序列化的对象未以 `__` 开头命名 — 进程重启后丢失
- 忘记实现 `process_initialize` — 模拟盘每日重启后全局变量未初始化
